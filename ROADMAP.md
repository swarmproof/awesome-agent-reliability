# awesome-agent-reliability — Roadmap

## Ongoing

- Monthly refresh: dead-link sweep, staleness check on `maintained` entries, add/cut pass. Each refresh lands as a dated `docs(refresh)` commit and updates the *Last refreshed* marker.
- Community PRs welcome — the quality bar and entry grammar live in [`docs/ARCHITECTURE.md`](./docs/ARCHITECTURE.md).

## Later (optional)

- Machine-readable entries (`data/entries.yaml`) as the source of truth, with the README generated from it (see ADR-005).
- A generated, searchable static site with section/tag filters, derived from the same data.
